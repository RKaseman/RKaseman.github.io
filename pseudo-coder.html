Some SharePoint code I worked on for my previous employer's Wiki site.

It was never what I wrote because SharePoint has the in-the-background habit of bloating and mangling code beyond all recognition. Even as an untrained amateur I was frustrated that what I put in was never what I got back and I Googled hoping for a way to prevent it. "Tough luck" was the answer. If I were tackling the same projects in the SharePoint environment today, I wonder how differently I'd approach it.

I enjoyed doing SharePoint work in spite of its flaws. It kept me dreaming of coding.

The code was copied directly from SharePoint's HTML editor into a Word document. When I was ready to add it to my portfolio I copied it from the Word document into VSCode, saw all of the warnings it generated and outdated, inline style tags it contained, and got to noting and fixing the worst offenders -- because I'll never push files that contain editor error warnings, and troubleshooting is a fun puzzle.
